# 🎯 设计团队智能排期系统 - 使用指南

## 系统概述

基于飞书多维表格的设计任务管理系统，提供：
- 🤖 智能任务分配（根据技能、经验、负载）
- 📅 自动排期计算（考虑紧急程度和任务复杂度）
- ⚠️ 超期预警（终稿时间 vs 截止时间）
- 👥 设计师资源管理（技能、经验、可用性）

## 飞书表格字段说明

### 基础信息字段

| 字段 | 类型 | 说明 |
|-----|------|------|
| 任务名称 | 文本 | 任务标题 |
| 任务描述 | 文本 | 详细需求说明 |
| 需求方 | 单选 | 产品/运营/市场/销售 |
| 需求方稿件截止时间 | 日期 | 最终交付时间 |

### 策划相关字段

| 字段 | 类型 | 说明 |
|-----|------|------|
| 策划人员 | 人员 | 提供策划内容的人员 |
| 策划初稿时间 | 日期时间 | 策划提交初稿时间 |
| 策划终稿时间 | 日期时间 | 策划提交终稿时间 |

### 设计师专属字段

| 字段 | 类型 | 说明 |
|-----|------|------|
| 设计师 | 人员 | 任务负责人 |
| 设计师开始时间 | 日期时间 | 设计师开始工作的时间 |
| 初稿时间 | 日期时间 | 设计师提交初稿的时间 |
| 终稿时间 | 日期时间 | 设计师提交终稿的时间 |

### 状态和预警

| 字段 | 类型 | 说明 |
|-----|------|------|
| 任务状态 | 单选 | 待分配/待完成/进行中/已完成/暂停 |
| 超期预警 | 文本 | 自动标记：⚠️ 超期 |

### 排期和优先级

| 字段 | 类型 | 说明 |
|-----|------|------|
| 当前排期紧急程度 | 单选 | S/A/B/C |
| 优先级 | 单选 | P0/P1/P2/P3 |
| 创建时间 | 日期时间 | 自动生成 |

## 智能排期逻辑

### 紧急程度计算

| 紧急程度 | 时间范围 | 标记 |
|---------|---------|------|
| S | 3天内 | 🔴 紧急 |
| A | 1周内 | 🟠 高 |
| B | 2周内 | 🟡 中 |
| C | 1月内 | 🟢 低 |

### 工期估算

根据任务类型和复杂度自动估算：

| 任务类型 | 简单 | 中等 | 复杂 |
|---------|-----|------|------|
| UI设计 | 2天 | 3天 | 4天 |
| UX设计 | 3天 | 5天 | 7天 |
| 插画 | 5天 | 7天 | 10天 |
| 动效 | 3天 | 4天 | 6天 |
| 品牌设计 | 7天 | 10天 | 15天 |

### 设计师推荐算法

考虑因素：
1. **技能匹配度**（40%）- 技能和专长匹配
2. **工作负载**（60%）- 当前任务数量
3. **经验等级**（参考）- 优先考虑高级设计师
4. **可用性**（过滤）- 排除休假中的设计师

### 超期预警

```
IF 终稿时间 > 需求方截止时间:
    警告 = "⚠️ 超期 X 天"
ELSE IF 剩余 < 2天:
    警告 = "⚡ 时间紧张，建议加快进度"
ELSE:
    警告 = "✅ 安全，剩余 X 天"
```

## 常见问题

### Q: 如何添加新设计师？

A: 编辑 `references/designers.json`，添加新的成员对象：

```json
{
  "name": "设计师F",
  "department": "设计部",
  "skills": ["UI设计", "Figma"],
  "specialties": ["移动端设计"],
  "experience_level": "中级",
  "tags": ["design", "ui"],
  "workload": 0,
  "availability": null
}
```

### Q: 如何更新设计师可用性？

A: 编辑对应设计师的 `availability` 字段：

```json
"availability": {
  "from": "2026-02-10",
  "to": "2026-02-15",
  "notes": "外部培训"
}
```

### Q: 工作负载如何自动更新？

A: 当任务成功分配给设计师后，其 `workload` 字段会自动 +1。任务完成后，需要手动编辑 JSON 文件减去对应数量。

### Q: 如何自定义工期估算？

A: 编辑 `scripts/intelligent_scheduler.py` 中的 `estimate_task_duration` 方法，修改 `base_duration` 字典。

### Q: 如何查看所有设计师的状态？

A: 运行：
```bash
python assign_task.py --members-file ../references/designers.json --list-members
```
