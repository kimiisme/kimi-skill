# 飞书任务自动分发技能

智能任务分配系统，基于团队成员技能自动分配任务并记录到飞书多维表格。

## ✨ 特性

- 🎯 **智能匹配** - 根据技能和角色自动匹配合适的团队成员
- ⚖️ **负载均衡** - 考虑工作负载，选择任务最少的成员
- 📊 **飞书集成** - 自动记录任务到飞书多维表格
- 🚀 **简单易用** - 自然语言交互，无需复杂配置
- 🔄 **实时更新** - 任务分配后立即更新飞书表格

## 📋 前置条件

### 必需配置

1. **飞书 MCP 服务器** - ✅ 已配置
   - 在 `~/.claude.json` 中配置
   - 状态：`feishu · ✓ connected`

2. **飞书应用权限** - ✅ 已配置
   - OAuth 登录已完成
   - 权限：`docs:doc`, `drive:drive`, `bitable:app`

3. **团队成员配置**
   - 编辑 `references/team_members.json`

## 🚀 快速开始

### 1. 配置团队成员

编辑 `references/team_members.json`：

```json
{
  "members": [
    {
      "name": "张三",
      "department": "Engineering",
      "skills": ["Python", "Backend", "API"],
      "tags": ["backend", "后端", "开发"],
      "workload": 0
    },
    {
      "name": "李四",
      "department": "Engineering",
      "skills": ["React", "Frontend", "TypeScript"],
      "tags": ["frontend", "前端", "开发"],
      "workload": 0
    }
  ]
}
```

### 2. 验证 MCP 连接

在 Claude Code 中运行：
```bash
/mcp
```

应该显示：`feishu · ✓ connected`

### 3. 开始使用

直接用自然语言描述需求：

```
分配一个后端任务：优化数据库查询性能
```

## 🎯 核心功能

### 1. 智能任务分配

根据任务类型和描述，自动匹配最合适的团队成员：
- 技能匹配（如 Python、React）
- 标签匹配（如 backend、frontend）
- 工作负载均衡（选择任务最少的成员）

### 2. 自动记录到飞书

通过飞书 MCP 自动创建多维表格记录：
- 使用用户身份（`useUAT: true`）
- 支持自定义字段
- 返回记录确认

### 3. 工作负载管理

- 自动跟踪每个成员的任务数量
- 任务分配后自动 +1
- 支持手动调整（编辑 JSON 文件）

## ❓ 常见问题

### Q: MCP 连接失败怎么办？

**A**:
1. 检查 `~/.claude.json` 配置是否正确
2. 运行 `/mcp` 查看连接状态
3. 重新登录：`npx -y @larksuiteoapi/lark-mcp login`
4. 重启 Claude Code

### Q: 找不到匹配的成员？

**A**:
1. 检查任务类型是否在成员的 skills 或 tags 中
2. 运行 `python scripts/assign_task.py --list-members` 查看所有成员
3. 调整技能/标签配置

### Q: 如何更新工作负载？

**A**:
- 任务成功分配后自动 +1
- 手动编辑 `references/team_members.json`
- 修改对应成员的 `workload` 字段

### Q: 可以记录到不同的表格吗？

**A**: 可以！在使用时指定新的表格地址
